# YasserOS Branding Guidelines

## Brand Identity

YasserOS has a clear, minimal identity that reflects its creator's engineering sensibility:

- **Modern but grounded** — clean design inspired by professional Linux distributions
- **Personal** — this is one person's OS; it can have personality
- **Arabic-influenced** — nods to the creator's heritage without being heavy-handed
- **Dark-first** — dark mode is the default (but light mode is available)

## Core Brand Values

1. **Precision** — every detail is intentional
2. **Minimalism** — no visual noise
3. **Authenticity** — no corporate polish; personal and honest
4. **Technical** — the aesthetic of someone who knows what they're doing

## Logo Concept

The YasserOS logo is the letter **Y** stylised to evoke:
- A circuit board trace or signal flow
- Arabic calligraphy influence on the Y stroke
- Clean, geometric construction

**Logo usage rules:**
- Minimum size: 24×24px (icon contexts)
- Clear space: at least 1× the logo height on all sides
- Do not rotate, distort, or apply effects to the logo
- Do not use the logo on low-contrast backgrounds

## Tagline

> "My machine. My rules. My OS."

This tagline may appear in:
- Boot splash (optional)
- About dialog (smaller text)
- GitHub README

Do not use the tagline in UI elements like panel or window titles.

## Tone of Voice

YasserOS communication (README, docs, dialogs) uses:
- Direct, confident statements
- Technical accuracy without unnecessary jargon
- A hint of personality (this is a personal project, not enterprise software)
- No marketing language or hype

Examples:
- Good: "YasserOS is built on Raspberry Pi OS. It is a personal hobby project."
- Bad: "YasserOS is the ultimate Raspberry Pi experience!"

## Application Identity

The Yasser Control Center branding:
- Name: `Yasser Control Center` (not `YasserOS Control Center`)
- Icon: Y logo variant, 48×48px
- About text includes hobby disclaimer
